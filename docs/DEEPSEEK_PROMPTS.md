# DeepSeek 审计提示词模板

> 版本：v9 · 状态：审计准备版

---

## 通用约束（适用于所有审计提示词）

**明确禁止**：
1. ❌ 不允许重写架构
2. ❌ 不允许改名（类名、方法名、文件名、目录名）
3. ❌ 不允许删除模块
4. ❌ 不允许牺牲真实功能去迎合测试
5. ❌ 不允许修改任何 `.py` 文件（只提供审计报告和建议）
6. ❌ 不允许引入新的目录或模块

**明确允许**：
1. ✅ 分析代码逻辑并提供改进建议
2. ✅ 评估边界条件和错误处理
3. ✅ 提出降本增效方案
4. ✅ 优化 prompt 设计建议
5. ✅ 提出指标和监控建议

## 模板 1：DEEPSEEK_AUDIT_NEWS_PIPELINE_V1

### 背景

FZQ-AI 是一个全球情报分析系统，核心组件是 `NewsPipeline`，负责从新闻输入到多维度分析（叙事/风险/情感/情景）的完整处理。当前版本 v8，已实现 JSON 解析 v2、字段完整性检查、并发执行、翻译 fallback 等增强。

### 审计目标

评估 `NewsPipeline` 的翻译逻辑、多维度分析逻辑、合并逻辑在所有边界条件下的表现一致性和鲁棒性。

## 使用指南

### 如何调用 DeepSeek 审计

1. **选择审计任务**：从 `DEEPSEEK_AUDIT_TASKS.md` 中选择任务（按优先级）
2. **准备上下文**：将对应文档作为 context 提供给 DeepSeek
3. **使用提示词模板**：从本文件选择对应提示词
4. **明确约束**：在提示词中重复声明"不改代码、不改架构、不改接口"
5. **接收输出**：按输出格式接收审计报告
6. **人工复核**：审计报告由人类复核后，决定是否实施建议

### 提示词优化技巧

1. **分段提供上下文**：不要一次性提供所有文档，按任务提供相关文档
2. **明确行号范围**：如果关注特定函数，提供代码行号范围
3. **要求对比**：要求 DeepSeek 提供"改进前 vs 改进后"的对比
4. **要求量化**：要求 DeepSeek 提供评分、百分比、成本估算等量化指标
5. **限制输出长度**：如果输出过长，要求分节输出

### 成本控制

| 审计任务 | 预估 token 消耗 | 预估成本（DeepSeek） | 建议 |
|----------|---------------|----------------------|------|
| 任务 1：NewsPipeline | ~50k | ~$0.05 | 分节审计（先翻译，再各维度） |
| 任务 2：News Intake | ~40k | ~$0.04 | 重点关注过滤和去重 |
| 任务 3：成本评估 | ~30k | ~$0.03 | 一次性完成 |
| 任务 4：Prompt 设计 | ~60k | ~$0.06 | 按模板分批审计 |
| 全部任务 | ~200k | ~$0.20 | 分 4-5 次调用 |

# DeepSeek 审计任务清单

> 版本：v9 · 状态：审计准备版
> 目标：为 DeepSeek 提供结构化、高价值的审计任务

---

## 审计原则

1. **不改代码**：DeepSeek 只提供审计报告、建议和风险评估，不直接修改代码
2. **不改架构**：审计范围限于当前架构内的优化，不引入新模块或新层次
3. **不改接口**：所有建议必须保持现有接口签名不变
4. **不改测试**：任何优化建议不得破坏 88 个现有测试

## 任务 1：NewsPipeline 逻辑一致性与鲁棒性审计

| 属性 | 值 |
|------|-----|
| **优先级** | P0（最高） |
| **审计目标** | 确保 NewsPipeline 的翻译、分析、合并逻辑在所有边界条件下表现一致 |
| **审计输入** | `docs/DATA_FLOW_PIPELINES.md`, `docs/SCHEMAS_MAP.md`, `fzq_ai/pipelines/real/news_pipeline.py` |
| **审计范围** | `_translate_if_needed()`, `_analyze_item()`, `_run_narrative()`, `_run_risk()`, `_run_sentiment()`, `_run_scenario()`, `run()` |
| **期望输出** | 1. 边界条件分析（空 items、空 text、超长 text、特殊字符）<br>2. 错误传播路径分析<br>3. 降级策略评估（是否充分、是否合理）<br>4. JSON 解析失败后的 fallback 质量评估<br>5. 并发安全分析（Semaphore + gather） |
| **风险等级** | 🔴 高（Pipeline 是核心，任何问题影响所有分析结果） |
| **建议交付** | 问题列表 + 风险等级 + 修复建议（每行代码标注） |

| 属性 | 值 |
|------|-----|
| **优先级** | P0（最高） |
| **审计目标** | 评估多源抓取策略的覆盖率、去重效果、相关性过滤精度 |

| **期望输出** | 1. 抓取覆盖率评估（30+ RSS 源是否足够？缺失哪些重要区域/语言？）<br>2. 去重策略评估（SHA-256 是否足够？是否需要语义去重？）<br>3. 相关性过滤评估（批量 10 条的精度 vs 成本 tradeoff）<br>4. 平衡报告有效性（平衡度评分是否真的能反映质量？）<br>5. 错误隔离评估（单源失败是否会影响其他源？） |

| 属性 | 值 |
